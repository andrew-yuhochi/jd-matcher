## Part 4 — Single-use token behavior (finding vs spike)

The research spike tested 8/8 URLs on a fresh inbox session. During
implementation, re-running the same fixture emails consumed those tokens.
Observation: Indeed pagead/clk URLs are single-use click tokens. In
production, the pipeline runs once per email batch (immediately on receipt)
so this is not a real-world limitation — it only affects re-testing with
the same fixture files. The correct validation approach is to reserve one
email set and run it exactly once, or use the offline unit tests (which
mock HTTP and are not affected by token expiry).

The AC #8 integration test is guarded by @pytest.mark.skipif(SKIP_LIVE="1")
and should only be run once against a fresh email batch. The results above
(97.1%) represent that first-run measurement.
